# レスポンシブスケールの実装戦略

日付: 2026-05-07
担当チーム: design

## 学んだこと

- 720px の narrow column レイアウトは「自然に縮む」ように見えて、視覚レビューで「レスポンシブ対応が入っていない」と即指摘される。`max-width: 720px` + `padding-inline` だけでは **font-size と section padding は狭幅でも変わらず**、375px 幅では明らかに窮屈に見える。
- グローバル側 (`design/globals.css`) で **h1〜h3 の font-size を一段下げる** + コンポーネント側で **section padding-block を `--space-7 → --space-6` に圧縮** という二層構造が効率的。グローバルだけだと個別調整できず、コンポーネントだけだと毎回 h タグの size を上書きする必要が出てしまう。
- ブレークポイントは **768px / 480px の 2 段階**で十分。1 段階だと「タブレット幅で h1 が大きすぎる」「スマホ幅でカード padding が広すぎる」のどちらかが破綻する。

## 失敗した点

- 最初の Skills 再デザイン spec ではレスポンシブを `< 480px` の 1 段階しか入れていなかった。他コンポーネントの spec も同様で、レビューで一括指摘されてから全コンポーネント横断で書き直した（commit 0f2cde9）。**最初から spec テンプレに「≥768 / 768〜480 / <480 の 3 行」を強制**しておけば手戻りなし。

## 次に活かす

- `design/specs/<X>.md` のレスポンシブ節は最低 3 行（≥768 / 768〜480 / <480）で埋める。空でも「変更なし」と明記する。
- 新コンポーネント追加時の確認チェックリスト: トークン参照・h タグ階層・**3 ブレークポイント**・aria-labelledby。
